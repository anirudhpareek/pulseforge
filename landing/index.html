Realtime charts that move like live systems, not screenshots.

PulseForge is a React chart component for streaming products. One canvas, smooth interpolation, and production-ready defaults for performance, readability, and control.

0 fps 0 points line mode

Getting started

Install with npm:

npm install @anirudhpareek/pulseforge

Why PulseForge

Most chart libraries are either heavy or rigid. They can look great on static data and fall apart under real stream conditions. PulseForge focuses on one core job: render changing values in a way that feels continuous, readable, and trustworthy.

  • Realtime rendering: single-canvas loop for low overhead
  • Line + candlestick modes for different reading styles
  • Decimation to preserve shape under dense streams
  • Reduced-motion + SSR-safe defaults for real apps
  • Benchmark and stress tooling before shipping

Quick usage

Drop in one component, set a stream, and tune max points for density.

import { PulseForge } from '@anirudhpareek/pulseforge';

<PulseForge
  mode="line"
  lineData={stream}
  width={900}
  height={320}
  maxPoints={700}
  showFps
/>

What it does

  • Realtime rendering with a single-canvas loop
  • Line and candlestick modes with consistent behavior
  • Decimation for dense, high-frequency streams
  • SSR-safe and reduced-motion aware defaults
  • Benchmark and stress tooling for production confidence

How it works

New values are interpolated into a continuous motion path instead of snapping frame-to-frame. This keeps trend direction readable and reduces visual noise during jittery updates.

Stress philosophy

A chart that only survives clean demo data is not production-ready. PulseForge is tested against:

  • Fast oscillations
  • Delayed and bursty updates
  • Sudden spikes and abrupt reversals
  • Reconnect gaps and catch-up windows
